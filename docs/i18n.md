# Internationalization (i18n) Architecture

The SmartDine frontend uses `react-i18next` for robust internationalization (i18n) to support multiple languages and prevent React DOM synchronization crashes caused by built-in browser translation tools (such as Google Translate).

## Why Native i18n?

Previously, when a user opened the application using a device configured to Hindi, their mobile browser automatically mutated the DOM to translate English text into Hindi using `<font>` tags. 
When React subsequently tried to conditionally update these nodes (e.g., during the search loading state), it resulted in a `NotFoundError: Failed to execute 'removeChild' on 'Node'` exception and crashed the application entirely.

By providing native Hindi translations, the browser detects that the application is natively rendering the user's preferred language and refrains from modifying the DOM, resolving the crash.

## Configuration Details

The i18n configuration strictly supports English (`en`) and Hindi (`hi`).

### Files

- `src/i18n.js`: The central configuration initialization file for `i18next`. It restricts supported languages and assigns default locales.
- `src/locales/en/translation.json`: English translation strings.
- `src/locales/hi/translation.json`: Hindi translation strings.

### Usage

To translate text in a React component, use the `useTranslation` hook:

```jsx
import { useTranslation } from 'react-i18next';

export function MyComponent() {
  const { t } = useTranslation();

  return (
    <div>
      <h1>{t('myComponent.title')}</h1>
    </div>
  );
}
```

### Constraints

> [!IMPORTANT]  
> The application is strictly configured to only support English and Hindi. Adding other languages requires updating the `supportedLngs` array in `src/i18n.js`.
